## Engineering Practices

Use the engineering practice skills to guide implementation and review across any stack:

- `/eng-guide` — unsure which skill to use? Start here
- `/eng-security-review` — injection, input validation, auth/authz, secrets (OWASP ASVS)
- `/eng-lgpd` — personal data, retention, anonymization, data subject rights (LGPD)
- `/eng-api-design` — REST conventions: versioning, pagination, idempotency, status codes
- `/eng-db-migration` — migration safety: breaking changes, locks, zero-downtime
- `/eng-testing-strategy` — test pyramid, when to use unit/integration/e2e, mock anti-patterns
- `/eng-observability` — structured logging, correlation IDs, what to log (LGPD-aware)
- `/eng-twelve-factor` — config, stateless processes, dev/prod parity, disposability
- `/eng-cicd-review` — quality gates, SAST/SCA, secrets in pipelines
- `/eng-error-handling` — retry, backoff, circuit breaker, timeout, DLQ, idempotency
- `/eng-performance-review` — N+1 queries, missing indexes, caching, slow endpoints
- `/eng-contract-testing` — consumer-driven contracts, OpenAPI schema validation
- `/eng-runbook` — writing operational runbooks for alerts and on-call procedures
